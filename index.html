System Overview

About Bayt AlKhyata Mobile App

The Bayt AlKhyata mobile application is a comprehensive platform for sewing and tailoring education, connecting learners with expert instructors and providing access to courses, products, and a vibrant community. The app integrates seamlessly with Odoo ERP v18 to provide a robust backend infrastructure.

Key Features
  • Authentication & User Management: Secure OTP-based registration and login
  • E-Learning Platform: Comprehensive courses, quizzes, and certifications
  • E-Commerce Integration: Product catalog, shopping cart, and secure checkout
  • Points & Rewards System: Loyalty program with referral codes and discounts
  • Community Features: Blog posts, discussions, and user interactions
  • Reels & Media: Short video content with engagement features
  • Support & Information: Help center, terms, and company information
Technical Architecture
  • Backend: Odoo ERP v18 with custom modules and REST API controllers
  • Authentication: JWT token-based authentication with refresh mechanisms
  • Security: HTTPS encryption, input validation, and rate limiting
  • Data Format: JSON request/response with consistent error handling
  • Mobile Optimization: Optimized for mobile networks and offline capabilities

Secure user authentication system with OTP verification, JWT tokens, and comprehensive profile management.

/api/v1/auth/send-otp

Error Handling & Response Guidelines

Comprehensive error handling guidelines and standardized response formats for consistent error reporting and debugging.

Standard Response Format

Success Response Structure

All successful API responses follow this consistent format:

{ "success": true, "message": "Operation completed successfully", "data": { // Actual response data }, "timestamp": "2024-01-16T12:00:00Z", "request_id": "req_abc123def456" }
Error Response Structure

All error responses follow this consistent format:

{ "success": false, "error": { "code": "VALIDATION_ERROR", "message": "The provided data is invalid", "details": { "field": "phone_number", "issue": "Invalid phone number format" } }, "timestamp": "2024-01-16T12:00:00Z", "request_id": "req_abc123def456" }

HTTP Status Codes

Success Codes (2xx)
200 OK: Request successful
201 Created: Resource created successfully
204 No Content: Successful deletion
Client Error Codes (4xx)
400 Bad Request: Invalid request data
401 Unauthorized: Authentication required
403 Forbidden: Access denied
404 Not Found: Resource not found
422 Unprocessable Entity: Validation failed
429 Too Many Requests: Rate limit exceeded
Server Error Codes (5xx)
500 Internal Server Error: Server error
502 Bad Gateway: Gateway error
503 Service Unavailable: Service down

Application-Specific Error Codes

Authentication & User Management Errors
AUTH_001: Invalid OTP code
AUTH_002: OTP expired
AUTH_003: Phone number already registered
AUTH_004: Invalid phone number format
AUTH_005: Account locked due to multiple failed attempts
PHONE_EXISTS: Mobile number already registered (use during registration)
PHONE_NOT_FOUND: Mobile number not found (use during login)
INVALID_OTP: Invalid OTP verification code
INVALID_PASSWORD: Incorrect password provided
E-Learning & Course Errors
COURSE_001: Course not found
COURSE_002: Already enrolled in course
COURSE_003: Course enrollment limit reached
COURSE_004: Insufficient permissions to access course
E-Commerce & Payment Errors
SHOP_001: Product out of stock
SHOP_002: Invalid product quantity
SHOP_003: Payment processing failed
SHOP_004: Cart is empty
Points & Rewards Errors
POINTS_001: Insufficient points balance
POINTS_002: Invalid referral code
POINTS_003: Referral code already used

Security & Development Best Practices

Authentication & Authorization
  • JWT Tokens: All API requests (except public endpoints) require valid JWT token in Authorization header
  • Token Format: Authorization: Bearer {jwt_token}
  • Token Expiry: Access tokens expire in 1 hour, refresh tokens in 30 days
  • Rate Limiting: 100 requests per minute per user, 1000 per hour
  • HTTPS Only: All API communications must use HTTPS encryption
Request Headers
Content-Type: application/json Authorization: Bearer {jwt_token} Accept: application/json User-Agent: BaytAlKhyata-Mobile/1.0.0 (iOS/Android) X-Request-ID: {unique_request_id}
Data Validation
  • Input Sanitization: All user inputs are sanitized and validated
  • Phone Numbers: Must be in international format (+966XXXXXXXXX)
  • Passwords: Minimum 8 characters with uppercase, lowercase, and numbers
  • File Uploads: Maximum 10MB, allowed types: JPG, PNG, PDF, MP4
Error Logging & Debugging
  • Request IDs: Each request includes unique ID for tracking
  • Error Logs: All errors logged with context and stack traces
  • Performance Monitoring: Response times and resource usage tracked
  • Debug Mode: Additional error details in development environment
