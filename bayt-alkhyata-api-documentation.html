System Overview

The Bayt AlKhyata mobile application integrates with Odoo ERP v18 through a comprehensive REST API system. This documentation provides complete specifications for all endpoints, authentication methods, and data structures required for seamless mobile-to-backend communication.

Mobile-First Design

Optimized for mobile app consumption with efficient data structures and minimal payload sizes.

Secure Authentication

OTP-based registration and JWT token authentication for secure user sessions.

Real-time Sync

Seamless data synchronization between mobile app and Odoo backend systems.

Modular Architecture

Clean separation of concerns with dedicated endpoints for each app module.

Technical Requirements
  • Odoo Version: v18 Community/Enterprise Edition
  • Protocol: HTTPS only (TLS 1.2+)
  • Authentication: JWT tokens with refresh mechanism
  • Data Format: JSON request/response bodies
  • Rate Limiting: 1000 requests per hour per user
  • API Versioning: /api/v1/ prefix for all endpoints
Core Odoo Modules Integration
  • res.users & res.partner: User management and authentication
  • website_sale: E-commerce functionality and product catalog
  • website_slides (eLearning): Course management and progress tracking
  • website_blog: Community posts and discussions
  • stock: Inventory management and product availability
  • payment: Payment processing and transaction handling
  • loyalty: Points system and referral management

Standard Response Format

All API responses follow a consistent structure:

{ "success": true, "message": "Operation completed successfully", "data": { // Response payload }, "meta": { "timestamp": "2024-01-15T10:30:00Z", "version": "v1", "pagination": { "page": 1, "limit": 20, "total": 150 } } }

Authentication & User Management APIs

Secure user authentication system with OTP verification, JWT tokens, and comprehensive profile management.

POST Send OTP for Registration

/api/v1/auth/send-otp
Purpose

Send OTP verification code to user's phone number before account creation. Validates phone uniqueness and generates secure 6-digit code.

Request Headers
Content-Type: application/json
Request Body
{ "phone": "+201234567890", "country_code": "EG" }
Success Response (200)
{ "success": true, "message": "OTP sent successfully", "data": { "otp_id": "otp_12345", "expires_in": 300, "retry_after": 60 } }
Odoo Integration
  • Models: Custom otp.verification model
  • Fields: phone, otp_code (hashed), expiry_time, attempts, created_date
  • Validation: Check phone uniqueness in res.partner
  • SMS Provider: Integration with Twilio/2Factor/Infobip
  • Security: Rate limiting (max 3 OTP per hour per phone)
Error Responses
PHONE_EXISTS (409): Phone number already registered
INVALID_PHONE (400): Invalid phone number format
RATE_LIMIT (429): Too many OTP requests
